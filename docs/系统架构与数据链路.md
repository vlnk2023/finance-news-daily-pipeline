# 系统架构与数据链路

更新时间：2026-05-07

本文档用于中文说明当前财经新闻数据管道的跨平台系统架构、平台职责、数据链路和后续扩展方向。

## 一、当前完成程度

当前系统已经完成最小可用闭环：

```text
Telegram 公开频道
  -> GitHub Actions 定时采集
  -> Supabase 存储
  -> Cloudflare Workers AI 翻译
  -> 规则版中文日报生成
  -> GitHub Pages 静态网页展示
  -> Hugging Face Space 手动调试
```

已完成：

- Telegram `/s/` 静态页采集器。
- Supabase 数据库表结构。
- GitHub Actions 自动批处理。
- Cloudflare Workers AI 翻译接入。
- `translations` 翻译缓存。
- `daily_digests` 日报表。
- GitHub Pages 静态前端。
- Hugging Face Space 手动采集控制台。
- 当前配置 10 个 Telegram 源。

下一阶段建议：

- 将规则版日报升级为 LLM 结构化中文摘要。
- 给新闻源增加质量权重和过滤策略。
- 前端增加来源筛选、日期筛选、关键词搜索。
- 将每次流水线运行状态写入 `pipeline_runs`。
- 增加失败通知，例如 Telegram Bot 或 Discord Webhook。

## 三、生产数据链路交互图

```mermaid
sequenceDiagram
    participant Cron as GitHub Actions Cron
    participant Registry as feed-registry.json
    participant TG as Telegram /s 页面
    participant Collector as Collector
    participant DB as Supabase
    participant Translate as translate.py
    participant CF as Cloudflare Workers AI
    participant Digest as generate_digest.py
    participant Web as GitHub Pages
    participant User as 用户浏览器

    Cron->>Registry: 读取启用频道列表
    Cron->>Collector: 启动采集任务

    loop 每个 Telegram 源
        Collector->>TG: GET https://t.me/s/channel
        TG-->>Collector: 返回公开 HTML
        Collector->>Collector: 解析消息、标准化、生成 guid
        Collector->>DB: upsert sources / news_items
    end

    Cron->>Translate: 启动翻译任务
    Translate->>DB: 查询 translation_status=pending 的 news_items

    loop 每条待翻译新闻
        Translate->>Translate: 语言识别
        alt 中文内容
            Translate->>DB: title_zh/summary_zh 直接复制，标记 translated
        else 非中文内容
            Translate->>DB: 查询 translations 缓存
            alt 缓存命中
                DB-->>Translate: 返回 translated_text
                Translate->>DB: 更新 news_items
            else 缓存未命中
                Translate->>CF: 调用 m2m100 翻译到中文
                CF-->>Translate: 返回中文译文
                Translate->>DB: 写入 translations 缓存
                Translate->>DB: 更新 news_items 为 translated
            end
        end
    end

    Cron->>Digest: 启动日报生成
    Digest->>DB: 读取当天 translated news_items
    Digest->>Digest: 分类、排序、生成 Markdown
    Digest->>DB: upsert daily_digests

    User->>Web: 打开日报页面
    Web->>DB: 使用 publishable key 读取 daily_digests
    DB-->>Web: 返回日报列表和 Markdown
    Web-->>User: 渲染中文日报
```

## 四、平台职责总览

| 平台 | 当前职责 | 关键配置 | 后续提升空间 |
|---|---|---|---|
| Telegram | 公开频道数据源 | 使用 `/s/` 静态页 | 增加更多频道、源质量评分 |
| GitHub Actions | 定时采集、翻译、生成日报 | Repository Secrets | 拆分 job、失败重试、运行日志 |
| Supabase | 主数据库 | SQL schema + service key | 搜索、RLS 细化、视图、索引 |
| Cloudflare Workers AI | 非中文翻译 | Account ID + API Token | 批量翻译、额度控制、fallback |
| GitHub Pages | 日报网页展示 | Pages source = GitHub Actions | 搜索、筛选、详情页 |
| Hugging Face Space | 手动调试控制台 | Gradio Space | 备用 API、手动回填、翻译兜底 |

## 五、各平台部署条件和参数

### 1. Telegram

使用方式：

```text
https://t.me/s/<channel>
```

新增源文件：

```text
src/config/feed-registry.json
```

验证命令：

```powershell
python scripts\fetch_finance_news_daily.py --url https://t.me/s/<channel> --limit 3 --json
```

关键配置：

```json
{
  "feed_id": "tg_example",
  "source_id": "tg_example",
  "source_name": "Example",
  "platform": "telegram",
  "url": "https://t.me/s/example",
  "language_hint": "en",
  "enabled": true,
  "collect": {
    "lookback_hours": 24,
    "max_items_per_run": 20,
    "timeout_ms": 12000,
    "retries": 2
  }
}
```

## 七、模块级流程图

### 采集模块

```mermaid
flowchart LR
    A[读取 feed-registry.json] --> B[过滤 enabled=true]
    B --> C[TelegramFetcher 请求 /s/ 页面]
    C --> D[TelegramParser 解析 HTML]
    D --> E[CollectionRunner 加 source/feed metadata]
    E --> F[lookback 过滤]
    F --> G[run 内去重]
    G --> H[Supabase upsert news_items]
```

### 翻译模块

```mermaid
flowchart LR
    A[读取 pending news_items] --> B[detect_language]
    B --> C{是否中文}
    C -- 是 --> D[复制 title/summary 到 zh 字段]
    C -- 否 --> E[计算 source_hash]
    E --> F{translations 缓存命中}
    F -- 是 --> G[复用 translated_text]
    F -- 否 --> H[Cloudflare Workers AI 翻译]
    H --> I[写 translations]
    G --> J[更新 news_items translated]
    I --> J
    D --> J
```

### 日报模块

```mermaid
flowchart LR
    A[读取当天 translated news_items] --> B[按关键词规则分类]
    B --> C[生成今日要点]
    C --> D[生成 Markdown 日报]
    D --> E[upsert daily_digests]
    E --> F[前端读取展示]
```

## 八、当前主要提升空间

### 采集侧

- 增加更多来源类型，例如 RSS、网页、API。
- 增加频道权重和黑名单关键词。
- 支持历史回填模式。
- 支持每个 feed 的独立运行频率。

### 翻译侧

- Cloudflare m2m100 增加批量调用。
- 增加失败重试。
- 增加 Gemini/Groq/HF Space fallback。
- 增加翻译质量评分。
- 对超长文本做分块翻译。

### 日报侧

- 当前是规则版，后续应接 LLM。
- 增加结构化栏目：
  - 宏观政策
  - 市场表现
  - 科技与 AI
  - 网络安全
  - 地缘风险
  - 外汇与商品
- 增加新闻重要性排序。
- 增加去重聚类，避免多个源重复报道同一事件。

### 前端侧

- 增加日期路由。
- 增加搜索。
- 增加来源筛选。
- 增加原始新闻列表。
- 增加 markdown 导出。

### 运维侧

- 写入 `pipeline_runs`。
- 每次 Actions 失败发送通知。
- 显示最近一次流水线状态。
- 显示各来源最近采集时间。
